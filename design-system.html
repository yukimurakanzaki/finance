Gap Closures — Decisions Made

G-01 · Safe-to-spend null guard CLOSED

computeSafeToSpend() returns null (not NaN / 0) when allowance is not set. All consumers render "—" for null. Guard check added before division.

G-02 · detectTransfers() O(n²) CLOSED

Move detectTransfers() to a Web Worker. Main thread posts the batch; worker returns flagged rows. Unblocks the UI on 150-row imports. Worker is inline (blob URL) — no build config change needed.

G-03 · RecurringItem next_due stale on import CLOSED

After import atomic write, a advanceNextDue() pass runs inside the same Dexie transaction — advances all RecurringItems whose next_due is within the imported date range. No separate step, no silent staleness.

G-04 · PIN salt missing → lockout CLOSED

On verifyPin(), if device_salt is missing: force PIN reset flow, not a lockout. New salt generated, user sets new PIN. Old data is still accessible (salt only affects PIN verify, not Dexie encryption).

G-05 · ISO week Monday-start CLOSED

useWorkweek() uses explicit ISO 8601 arithmetic: Monday = (day + 6) % 7 === 0. Never uses getDay() raw. Unit-tested for Sun/Mon boundary on both hemispheres.

G-06 · Transfer matcher greedy one-to-one CLOSED

Matcher iterates out-rows, for each finds the earliest unmatched in-row with same amount ± 1 day. Marks both consumed. Second in-row with same amount is left unmatched and imports as normal income. Tested with 2 × 500K same-day scenario.

G-07 · Account.current_balance staleness CLOSED

Decision: drop Account.current_balance as a stored field. Always derive from transaction sum. Cash / wallet accounts use a manual_balance_override with last_updated_at — used when no transaction history exists for that account.

G-08 · Restore prompt — export first CLOSED

Restore modal: Step 1 = "Export current data first" with a one-tap export button. Step 2 = confirmation + file picker. Cannot reach file picker without seeing the export prompt. Prevents irrecoverable overwrites.

G-09 · reconcileInProgress flag persistence CLOSED

Flag persisted to localStorage (reconcile_in_progress: true) at parse start, cleared on confirm or cancel. On cold open, if flag is set, show "You had an import in progress — it was lost. Start again?" — no silent FAB appearance.

G-10 · last_exported_at in Dexie CLOSED

Move last_exported_at to an AppSettings Dexie table (single-row key-value store) so it survives localStorage clears alongside the main data.

G-11 · Gold revaluation prompt CLOSED

Home screen: if the gold asset's last_valued_at > 30 days, show a one-line inline nudge "Gold last priced 34 days ago — update?" with a direct tap-to-edit. Dismissable per session.

G-12 · FI engine pipe source CLOSED

computeFIDate() derives pipeMonthly from the sum of active pay_yourself_first RecurringItems — not from IncomeEvent. IncomeEvent is the historical record; RecurringItems are the live source of truth for the projection.

G-13 · iOS "Add to Home Screen" install instruction CLOSED

On iOS Safari (detect via navigator.standalone === false && /iphone/i.test(navigator.userAgent)), show a bottom banner: "Install: tap ⎋ Share → Add to Home Screen" with a dismiss. Shown once per session until installed.

Copywriting Guidelines + Copy Bank
Voice Rules — 5 constraints
Direct, no filler. "Rp 58k today" not "You currently have Rp 58,000 available to spend today!"
Show the math. Always expose the calculation. Adi is a PM — he wants to see how the number was built, not just the output.
No verdicts. "This delays FI by ~4 days" not "This is a bad purchase." Present the trade-off; let him decide.
Amber informs, never scolds. "Ran a little hot Mon–Tue" not "You overspent!" Neutral, factual, forward-looking.
No emojis. No exclamations. This is a dashboard, not a social app. Reserve exclamation marks for zero occurrences.

Design Rules — Developer Reference

Colour Rules

  • Amber is the only accent. No blue buttons, no green CTAs.
  • Red does not exist in the app. Debt, errors, and warnings use slate or amber.
  • Lane colours categorise only — never status signals.
  • Background elevation: deeper = more background, not more prominent.
  • Text on amber background (--amber) must use #000 (black). Contrast ≥ 4.5:1.
  • Never use raw hex values in components. Always use the token var().

Typography Rules

  • All currency/number values: font-family: var(--font-mono).
  • Hero numbers (Rp 58k, Rp 80M): 40–52px, weight 700, letter-spacing ≤ −1.5px.
  • Section labels: 9.5px, uppercase, 0.7px letter-spacing, weight 700, ink-3.
  • No italics. No underlines except interactive links.
  • Never justify text. Left-align all copy.
  • Amounts: right-align in tables/rows. Labels: left-align.

Spacing + Touch

  • Minimum tap target: 44×44px (iOS HIG). PIN keys: 56px height.
  • Screen horizontal padding: 20px left + right.
  • Card padding: 14px vertical, 16px horizontal.
  • Card border-radius: 14px (lg). Sheet: 22px top corners.
  • FAB: 52×52px, border-radius 16px. Bottom: 72px (above tab bar).
  • Tab bar height: 58px. Bottom inset: 10px for home indicator.

Motion + Feedback

  • Number updates (safe-to-spend): counter animation, 300ms ease-out.
  • Sheet open: slide up 250ms cubic-bezier(0.32,0.72,0,1).
  • PIN wrong attempt: horizontal shake, 4 cycles, 50ms each, ±8px.
  • Toast: fade in 200ms, hold 3s, fade out 200ms. Bottom of screen.
  • No skeleton loaders — data is local (IndexedDB), reads are synchronous.
  • FAB press: scale(0.93), 80ms. Release: scale(1), 120ms ease-out.

Amber State Rules

  • Amber background card: use --amber-surface + --amber-border only.
  • Text on amber-surface: use --amber-text (#fbbf24), not --amber (#f0a500).
  • Amber state never blocks action. Always informational.
  • Never show a negative number in the safe-to-spend gauge. Clamp to 0 and show amber banner.
  • The amber FAB (primary action) uses --amber background with #000 text.

What to Never Build

  • No red anywhere in the UI — not for errors, not for debt, not for negative deltas.
  • No sell button, no sell flow, no sell confirmation — anywhere.
  • No real-time market data, no price tickers, no live fund NAVs.
  • No push notifications for market movement or portfolio value.
  • No "you should cut this" suggestions. Decision lens shows cost, not verdict.
  • No negative currency display on safe-to-spend — always clamp to 0 + banner.
